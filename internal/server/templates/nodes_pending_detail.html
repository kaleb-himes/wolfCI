Cert material

The agent authenticates to the server via mTLS. Copy two files onto the remote machine, under /etc/wolfci-agent/ (matching the --cert-dir in the command above):

  1. ca.pem - the wolfCI server's CA certificate (config-files/wolfci-ca.pem on the server host). The agent uses it to verify the server's identity.
  2. agent.pem + agent.key - the agent's own certificate + private key, signed by the same CA. Generate them on the server with wolfci-ctl agent-cert mint --name {{ .Name }} (prints the cert + key to stdout) and copy the output onto the remote machine.

Permission: the /etc/wolfci-agent/ directory should be mode 0700 and owned by the user running wolfci-agent; the private key file should be mode 0600.
