Pick the job shape. Each option drops you on the same job-edit form afterwards; the choice you make here seeds which fields you'll typically reach for.

Inline script, simple job

A single-stage job whose steps are shell commands typed inline. Closest to a Jenkins Freestyle Project. Best for small builds, one-shot scripts, and anything that does not need parallel stages.

More complex, multi-step job

A pipeline-driven job with stages, parallel branches, and the Jenkinsfile-style DSL. Closest to a Jenkins Pipeline. Best for builds with setup / build / test / deploy stages or work that fans out across labeled nodes.

Most complex, multi-path, multi-step job

A pipeline that fetches its script from source control and fans across multiple branches automatically. Closest to a Jenkins Multibranch Pipeline. Best for repositories where every release branch should build the same Jenkinsfile.
